Home work #1
  • git config --global user.name - позволяет изменить имя пользователя
  • git config --global user.email - позволяет изменить почту пользователя
  • git init - эта команда создаёт в текущем каталоге новый подкаталог с именем .git, содержащий все необходимые файлы репозитория — структуру Git репозитория
  • git status - отображает состояние рабочего каталога и раздела проиндексированных файлов. С ее помощью можно проверить индексацию изменений и увидеть файлы, которые не отслеживаются Git
  • git add - добавляет содержимое рабочего каталога в индекс (staging area) для последующего коммита. По умолчанию git commit использует лишь этот индекс, так что вы можете использовать git add для сборки слепка вашего следующего коммита
  • git commit - это команда Git для записи индексированных изменений в репозиторий
  • git log - перечисляет коммиты, сделанные в репозитории в обратном к хронологическому порядке
  • git checkout - используется для переключения веток и выгрузки их содержимого в рабочий каталог
  • git branch - это своего рода "менеджер веток". Она умеет перечислять ваши ветки, создавать новые, удалять и переименовывать их
  • git push - используется для публикации выгружаемых локальных изменений в центральном репозитории
  • git pull - используется для извлечения и загрузки содержимого из удаленного репозитория и немедленного обновления локального репозитория этим содержимым
  • git merge - используется для слияния одной или нескольких веток в текущую. Затем она устанавливает указатель текущей ветки на результирующий коммит
  • git clone - клонирует репозиторий в новый каталог, создает ветви дистанционного отслеживания для каждого
  • .gitignore - это простой текстовый файл, в каждой строке которого содержится шаблон файла или каталога, который необходимо проигнорировать
